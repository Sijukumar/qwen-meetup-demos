## Key Components

### 1. Qwen3-Omni Model Specifications
| Aspect | Details |
|--------|---------|
| **Model Name** | `qwen3-omni-flash` |
| **Input Modalities** | Text, Image, Audio, Video |
| **Output Modalities** | Text + Audio (simultaneous) |
| **Streaming** | Mandatory (`stream=True`) |
| **Audio Format** | PCM (converted to WAV) |
| **Sample Rate** | 24kHz |
| **Bit Depth** | 16-bit |
| **Channels** | Mono |

### 2. Available Voices
| Voice | Description |
|-------|-------------|
| `Cherry` | Default female voice |
| `Dylan` | Male voice |
| `40+ others` | Various voice options |

### 3. Operating Modes
| Mode | Modalities | Use Case |
|------|-----------|----------|
| **Standard** | `["text", "audio"]` | Conversational AI with voice |
| **Deep Thinking** | `["text"]` | Complex reasoning (no audio) |

### 4. API Request Structure
```python
{
    "model": "qwen3-omni-flash",
    "messages": [
        {
            "role": "user",
            "content": [
                {"type": "image_url", "image_url": {"url": "data:image/jpeg;base64,..."}},
                {"type": "text", "text": "Describe this image"}
            ]
        }
    ],
    "modalities": ["text", "audio"],
    "audio": {
        "voice": "Cherry",
        "format": "wav"
    },
    "stream": True,
    "extra_body": {
        "enable_thinking": False
    }
}
```

### 5. Response Structure
```python
# Streaming chunks
chunk.choices[0].delta.content  # Text output
chunk.choices[0].delta.audio    # Audio data

# Audio data format
{
    "data": "base64_encoded_pcm_data"
}
```

## Usage Flow

```
1. Start: python qwen_omni_demo.py
2. Select mode:
   - [1] Text chat
   - [2] Image + text
   - [3] Toggle thinking mode
   - [4] Change voice
   - [5] Quit
3. Enter prompt (and image path if selected)
4. Wait: Streaming response begins
5. See: Text appears in real-time
6. Hear: Audio plays automatically (if enabled)
7. Repeat: Continue conversation
```

## Critical Constraints

| Constraint | Impact |
|------------|--------|
| **Streaming Mandatory** | Cannot use non-streaming mode |
| **Thinking Disables Audio** | Deep thinking = text only |
| **Single Modality Pair** | One image OR audio per message (not both) |
| **PCM Conversion** | API returns raw PCM, must convert to WAV |

## Error Handling

| Error Type | Handling |
|------------|----------|
| **Missing API Key** | Exit with error message |
| **Image Not Found** | Print error, return to menu |
| **API Call Failed** | Print exception details |
| **Audio Conversion Failed** | Save raw PCM as fallback |
| **Audio Playback Failed** | Display manual play instructions |
| **Keyboard Interrupt** | Graceful exit |

## Timing Characteristics

| Phase | Typical Duration |
|-------|-----------------|
| API Connection | < 1 second |
| First Token | 1-3 seconds |
| Full Response | 5-15 seconds |
| Audio Generation | Concurrent with text |
| Audio Conversion | < 1 second |
| Audio Playback | Duration of speech |
| **Total Latency** | **< 3 seconds** (first response) |

## Security Considerations

1. **API Key**: Stored in `.env` file, never hardcoded
2. **Image Data**: Base64 encoded, sent via HTTPS
3. **Audio Output**: Saved locally, not transmitted
4. **No Persistent Storage**: Conversation not saved between sessions
